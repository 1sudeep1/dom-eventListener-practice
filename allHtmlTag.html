Semantic HTML Tags and their uses

Basic HTML Structure

  1. !DOCTYPE
    Defines the document type and HTML version.
  2. html
    Root element of an HTML document.
  3. head
    Contains meta-information about the document (e.g., title, meta tags, links to scripts and styles).
  4. title
    Sets the title of the document, shown in the browser's title bar or tab.
  5. meta
    Provides metadata such as descriptions, keywords, author, and character set.
  6. link
    Defines the relationship between the current document and an external resource (e.g., stylesheets).
  7. style
    Contains internal CSS styles.
  8. script
    Embeds or references executable JavaScript code.
  9. body
    Contains the content of the document, visible to users.

Content Sectioning

  1. header
    Represents introductory content or a set of navigational links.
  2. nav
    Represents a section with navigation links.
  3. section
    Defines a section in a document.
  4. article
    Represents a self-contained piece of content.
  5. aside
    Represents content tangentially related to the content around it.
  6. footer
    Represents a footer for its nearest sectioning content or sectioning root element.
  7. main
    Represents the main content of the document.

Text Content

  1. h1 to h6
    Define HTML headings, h1 being the highest (or most important) level and h6 the least.
  2. p
    Represents a paragraph.
  3. blockquote
    Indicates a section that is quoted from another source.
  4. ol
    Defines an ordered list.
  5. ul
    Defines an unordered list.
  6. li
    Defines a list item.
  7. dl
    Defines a description list.
  8. dt
    Defines a term in a description list.
  9. dd
    Describes a term in a description list.
  10. figure
    Specifies self-contained content, often with a caption.
  11. figcaption
    Provides a caption for a figure element.
  12. div
    Defines a division or section in an HTML document.
  13. span
    Used for grouping and applying styles or scripts to inline elements.
  14. a
    Defines a hyperlink.
  15. em
    Indicates emphasized text.
  16. strong
    Indicates important text.
  17. small
    Represents side-comments and small print.
  18. s
    Specifies text that is no longer correct, accurate, or relevant.
  19. cite
    Represents a reference to a creative work.
  20. q
    Indicates a short inline quotation.
  21. code
    Represents a fragment of computer code.
  22. pre
    Represents preformatted text.
  23. br
    Inserts a single line break.
  24. hr
    Represents a thematic break between paragraph-level elements.
  25. mark
    Represents text highlighted for reference purposes.
  26. b
    Represents text stylistically different from normal text, without conveying extra importance.
  27. i
    Represents a span of text in an alternate voice or mood.
  28. u
    Represents text that should be stylistically different from normal text, such as misspelled words.
  29. sub
    Specifies subscripted text.
  30. sup
    Specifies superscripted text.
  31. kbd
    Represents user input, typically keyboard input.
  32. var
    Represents a variable in programming or mathematical expressions.
  33. samp
    Represents sample output from a computer program.
  34. time
    Represents a specific period in time.

Forms

  1. form
    Represents a document section containing interactive controls for submitting information.
  2. input
    Represents an input control.
  3. textarea
    Represents a multi-line plain-text editing control.
  4. button
    Represents a clickable button.
  5. select
    Represents a control that provides a menu of options.
  6. option
    Defines an option in a select list.
  7. optgroup
    Groups options in a dropdown list.
  8. label
    Represents a caption for an item in a user interface.
  9. fieldset
    Groups related elements in a form.
  10. legend
    Represents a caption for a fieldset.
  11. datalist
    Specifies a list of pre-defined options for an input element.
  12. output
    Represents the result of a calculation.

Tables

  1. table
    Represents tabular data.
  2. caption
    Represents the title of a table.
  3. thead
    Groups the header content in a table.
  4. tbody
    Groups the body content in a table.
  5. tfoot
    Groups the footer content in a table.
  6. tr
    Represents a row of cells in a table.
  7. th
    Represents a header cell in a table.
  8. td
    Represents a cell in a table.

Embedded Content

  1. img
    Embeds an image in the document.
  2. audio
    Embeds sound content.
  3. video
    Embeds video content.
  4. source
    Specifies multiple media resources for video and audio elements.
  5. track
    Specifies text tracks for video and audio elements.
  6. iframe
    Embeds another HTML page within the current page.
  7. embed
    Embeds external content or applications.
  8. object
    Represents an external resource, such as an image, HTML document, or plugin.
  9. param
    Defines parameters for an object element.
  10. map
    Defines an image map (a clickable area within an image).
  11. area
    Defines an area inside an image map.
  12. canvas
    Used for drawing graphics via scripting (usually JavaScript).
  13. svg
    Defines vector-based graphics.

Scripting

  1. script
    Embeds or references executable JavaScript code.
  2. noscript
    Defines an alternate content for users that do not support client-side scripts.

Interactive Elements

  1. details
    Represents a disclosure widget from which the user can obtain additional information or controls.
  2. summary
    Provides a summary, caption, or legend for a details element.

Web Components

  1. template
    Holds client-side content that is not to be rendered when the page loads but may be instantiated later during runtime using JavaScript.
  2. slot
    Placeholder inside a web component that users can fill with their own markup.
